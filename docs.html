常用工具命令

以下整理了我们日常使用频率最高的生信工具的核心命令，更多细节请参考视频课程。

FastQC：测序数据质控

# 单个样本质控 fastqc -o output_dir input.fastq.gz # 多线程批量质控 fastqc -t 8 -o output_dir *.fastq.gz # 合并报告 multiqc output_dir/

BWA：序列比对

# 建立参考基因组索引 bwa index reference.fa # 双端比对 bwa mem -t 16 -R "@RG\tID:S1\tSM:S1\tLB:S1\tPL:ILLUMINA" \ reference.fa S1_R1.fq.gz S1_R2.fq.gz | samtools sort -@ 8 -o S1.sorted.bam

GATK：变异检测

# 标记重复 gatk MarkDuplicates -I input.bam -O marked.bam -M metrics.txt # 调用变异 gatk HaplotypeCaller -R reference.fa -I marked.bam -O variants.vcf.gz # 过滤 gatk VariantFiltration -R reference.fa -V variants.vcf.gz \ --filter-expression "QD < 2.0" --filter-name "LowQD" -O filtered.vcf.gz

samtools：BAM 文件处理

# 索引 samtools index input.bam # 排序 samtools sort -@ 8 -o sorted.bam input.bam # 比对统计 samtools flagstat input.bam # 深度统计 samtools depth -a input.bam | awk '{sum+=$3} END {print sum/NR}'

分析流程模板

以下是几个常用的端到端分析流程模板，可以作为新项目的起点。

WGS 全基因组重测序流程

#!/bin/bash # 全基因组重测序标准流程 REF="/data/ref/hg38.fa" SAMPLE=$1 THREADS=16 # Step 1: 质控 fastqc -t $THREADS -o qc/ ${SAMPLE}_R1.fq.gz ${SAMPLE}_R2.fq.gz # Step 2: 比对 bwa mem -t $THREADS -R "@RG\tID:$SAMPLE\tSM:$SAMPLE\tLB:$SAMPLE\tPL:ILLUMINA" \ $REF ${SAMPLE}_R1.fq.gz ${SAMPLE}_R2.fq.gz | \ samtools sort -@ $THREADS -o ${SAMPLE}.sorted.bam # Step 3: 标记重复 gatk MarkDuplicates -I ${SAMPLE}.sorted.bam -O ${SAMPLE}.dedup.bam -M ${SAMPLE}.metrics.txt # Step 4: 调用变异 gatk HaplotypeCaller -R $REF -I ${SAMPLE}.dedup.bam -O ${SAMPLE}.g.vcf.gz -ERC GVCF # Step 5: 联合分型 gatk GenotypeGVCFs -R $REF -V ${SAMPLE}.g.vcf.gz -O ${SAMPLE}.vcf.gz # Step 6: 注释 table_annovar.pl ${SAMPLE}.vcf.gz humandb/ -buildver hg38 \ -out ${SAMPLE} -protocol refGene,clinvar_20240611 -operation g,f

RNA-Seq 转录组分析流程

#!/bin/bash # RNA-Seq 标准流程（hisat2 + featureCounts） REF_INDEX="/data/ref/hisat2_index" GTF="/data/ref/hg38.gtf" SAMPLE=$1 # Step 1: 比对 hisat2 -p 16 -x $REF_INDEX \ -1 ${SAMPLE}_R1.fq.gz -2 ${SAMPLE}_R2.fq.gz \ -S ${SAMPLE}.sam # Step 2: 转换排序 samtools sort -@ 8 -o ${SAMPLE}.bam ${SAMPLE}.sam && rm ${SAMPLE}.sam # Step 3: 计数 featureCounts -T 8 -p -a $GTF -o ${SAMPLE}.counts ${SAMPLE}.bam # Step 4: R 中使用 DESeq2 做差异分析（另起脚本）

最佳实践

1. 文件组织规范

建议按以下结构组织项目文件，便于团队协作与归档：

project/ ├── raw_data/ # 原始测序数据（只读） ├── qc/ # 质控结果 ├── alignment/ # BAM 文件 ├── variant/ # 变异结果 ├── annotation/ # 注释结果 ├── scripts/ # 分析脚本 ├── results/ # 最终结果与图表 └── README.md # 项目说明

2. 资源使用建议

  • 共享服务器：单任务建议线程数不超过 16，内存不超过 64G
  • 大型分析任务请提前申请独享服务器
  • 中间文件（如 SAM）请及时清理或转换为压缩格式（BAM）
  • 使用 nohup 或 screen 保持长任务运行

3. 数据备份策略

💡 推荐： 采用「3-2-1」原则：3 份拷贝，2 种介质，1 份异地。 我们的云服务器自带备份，但重要原始数据建议本地额外保存。

常见问题

Q: 如何申请云服务器试用账号？

A: 请添加微信客服，提供姓名与单位信息，我们会为新用户开通 3 天免费试用账号。

Q: 服务器到期后数据会被清空吗？

A: 不会。账号到期后我们会保留数据 30 天，期间续费可恢复全部内容；超过 30 天未续费才会清理。

Q: 可以自行安装软件吗？

A: 共享服务器只能在个人目录下安装；独享服务器拥有 root 权限，可自由安装任何软件。

Q: 是否支持 GPU 计算？

A: 我们有 GPU 服务器可供选购，配备 NVIDIA 高性能卡，适合 AI 模型训练与深度学习推理任务。

Q: 培训证书可用于职称评定吗？

A: 我们颁发的是企业培训证书，部分单位可作为继续教育学时认定，具体请咨询所在单位人事部门。

Q: 视频课程可以分享给同事观看吗？

A: 课程仅授权给购买者本人观看，未经授权分享将影响后续续费与服务支持。如团队需要，建议团报享受优惠。
